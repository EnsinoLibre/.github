# Contributing to EnsinoLibre

Thank you for wanting to help. This guide applies across every repository in the [EnsinoLibre](https://github.com/ensinolibre) organisation. Some repos add their own `CONTRIBUTING.md` with detail specific to that repo. Read that one first if it exists, and use this one as the shared baseline.

You do not need to be a developer to contribute. Worksheets, translations, documentation fixes, and bug reports are all real contributions, and we treat them that way.

## Ways to contribute

- **Share a worksheet.** Fork [`examples`](https://github.com/ensinolibre/examples), add your worksheet under the right subject and grade folder, and open a pull request. See that repo's `CONTRIBUTING.md` for a walkthrough written for teachers.
- **Add or improve a translation.** Head to [`translations`](https://github.com/ensinolibre/translations). Portuguese and Spanish are first-class locales, and every other language is welcome.
- **Fix or improve documentation.** Open a pull request against [`docs`](https://github.com/ensinolibre/docs). Small clarity fixes are always welcome.
- **Report a bug or suggest a feature.** Open an issue in the repo it belongs to, using the issue templates. If you are not sure which repo, start at [`core`](https://github.com/ensinolibre/core) and we will help you route it.
- **Write code.** Look for issues labeled [`good first issue`](https://github.com/ensinolibre/core/labels/good%20first%20issue) across the org's repos.

## Before you start

1. Read the [Code of Conduct](CODE_OF_CONDUCT.md).
2. Check open issues and pull requests so you are not duplicating work.
3. For anything larger than a small fix, open an issue first to talk through the approach before you spend time on a pull request.

## Making a pull request

1. Fork the repo and create a branch from `main`.
2. Make your change. Keep pull requests focused on one thing.
3. Fill in the pull request template, including what you changed and why.
4. A maintainer will review, ask questions if needed, and merge once it is ready.

## Code contributions

- Match the existing style and structure of the repo you are working in.
- Add or update tests when you change behavior, if the repo has a test suite.
- Keep commits and pull requests small enough to review in one sitting.

## Getting help

If you are stuck, ask in [GitHub Discussions](https://github.com/orgs/ensinolibre/discussions). No question is too basic. Many of our contributors are teachers making their first ever pull request, and we want that experience to go well.
