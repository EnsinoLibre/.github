# Security Policy

## Reporting a vulnerability

If you find a security issue in any EnsinoLibre repository, please do not open a public issue. Email **security@ensinolibre.org** with:

- The repository and version affected
- A description of the issue and its potential impact
- Steps to reproduce, if you have them

We will acknowledge your report within 5 business days and keep you updated as we investigate and fix the issue. We will credit you in the fix, unless you would rather stay anonymous.

## Supported repositories

Security reports are welcome for any repository under the [EnsinoLibre](https://github.com/ensinolibre) organisation, in particular [`core`](https://github.com/ensinolibre/core) (the application handling worksheet data and sharing links) and [`blocks`](https://github.com/ensinolibre/blocks) (the grading logic).

## Scope

This policy covers vulnerabilities in our own code. If you find an issue in a third-party dependency we use, please also report it upstream to that project.
